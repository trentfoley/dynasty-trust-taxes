Create `fill_pdf.py` at the project root. It must implement the following behavior exactly (per locked decisions):

Interface:
  python fill_pdf.py --fields <json_file> --form <blank_pdf> [--output <path>]
  --output default: "output/2025_1041_filled.pdf"

Behavior:
1. Parse args with argparse. --help must work cleanly.
2. Load field_values from --fields JSON file ({"field_name": "value"}).
3. Open --form with PdfReader. Call reader.get_fields() to get known_fields.
4. Hard-fail validation: for any key in field_values that is NOT in known_fields, print to stderr:
   "ERROR: field not found in AcroForm: '<field_name>'"
   Then sys.exit(1). Check ALL unknown fields before exiting (collect, then exit once).
5. Checkbox translation: for each field in field_values where the caller provided "Yes":
   - Look up the field in reader.get_fields()
   - Read states = field.get("/_States_", [])
   - Find the first state that is not "/Off" — strip the leading "/" to get the On-state string
   - Replace "Yes" with that On-state string in the fields dict passed to update_page_form_field_values
   (For "Off" values, pass "Off" as-is — pypdf handles this correctly)
6. Create PdfWriter, clone reader: writer.clone_reader_document_root(reader)
7. Fill all pages in one call:
   writer.update_page_form_field_values(page=None, fields=field_values, auto_regenerate=True)
8. Create output directory if it doesn't exist: os.makedirs(os.path.dirname(args.output) or ".", exist_ok=True)
9. Write output PDF with: writer.write(f)
10. Print: "Filled PDF written to: {args.output}"

Imports needed: argparse, json, sys, os, from pypdf import PdfReader, PdfWriter

Do NOT include any tax bracket values, rate constants, or EIN/trust data in fill_pdf.py.
The script must know nothing about tax math — it is a dumb file writer.

After writing, run: /c/ProgramData/miniconda3/python fill_pdf.py --help
