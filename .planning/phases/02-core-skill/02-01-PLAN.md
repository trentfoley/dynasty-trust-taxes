CSV structure (2025/XXXX-X123.CSV):
- Section headers: "Form 1099DIV", "Form 1099INT", "Form 1099 B"
- 1099-DIV: rows keyed by box number (1a, 1b); amount in column 2, total in column 3
- 1099-INT: rows keyed by box number (1); amount in column 3
- 1099-B: 23-column header row followed by 2 data rows
  - Column 0: description ("296.00 VANGUARD..." or "1.35 VANGUARD...")
  - Column 1: date acquired (may be "Various")
  - Column 2: date sold
  - Column 3: proceeds
  - Column 4: cost basis
  - Column 6: wash sale loss disallowed
  - Column 7: term ("Long term" or "Short term")
  - Column 8: form8949_code ("D" or "A")
  - Column 11: covered status ("Covered")

<task type="auto">
  <name>Task 2: Create fill_1041.py with arg parsing and CSV parser</name>
  <files>fill_1041.py</files>
  <action>
Create fill_1041.py at the project root implementing the skeleton and CSV parsing layer. The script uses only stdlib: argparse, csv, json, subprocess, sys, pathlib.

Python executable path: /c/ProgramData/miniconda3/python (NOT python3).

Implement these functions:

parse_args(): Load config defaults first (from config/2025.json paths section), then build argparse with --year YYYY (default: 2025), --csv PATH (default: cfg paths csv_input), --output-dir PATH (default: cfg paths output_dir), --dry-run flag. Returns namespace.

load_config(year): Load and return json from config/{year}.json. Hard-fail with clear message if file not found.

load_fields(year): Load and return json from config/{year}_fields.json. Hard-fail if not found.

parse_dollar(s): Strip quotes, $, commas; return float. Return 0.0 for empty strings.

parse_csv(csv_path): State-machine CSV parser using csv.reader. Returns dict with keys: div_1a (float), div_1b (float), int_box1 (float), transactions (list of dicts).

Each transaction dict: description, date_acquired (may be "Various"), date_sold, proceeds (float), cost (float), wash_sale (float), term (str lowercased: "long term" or "short term"), form8949_code (str: "D" or "A"), covered (bool).

Section detection: check row[0].strip('"').strip() against 'Form 1099DIV', 'Form 1099INT', 'Form 1099 B'. Blank rows (no non-empty cell) reset section to None.

For 1099-DIV: box is row[0] stripped. Box 1a total is in row[3]; box 1b amount is in row[2].
For 1099-INT: box is row[0] stripped. Box 1 total is in row[3].
For 1099-B: after section header, skip the column header row (set col_headers flag), then parse each subsequent data row using: proceeds=row[3], cost=row[4], wash_sale=row[6], term=row[7].lower(), form8949_code=row[8].strip('"'), covered=(row[11].strip('"').lower()=='covered').

print_header(args, cfg): Print a 4-line header showing CSV input, blank form, output dir, and mode (DRY RUN or LIVE).

main():
1. Load config (year=2025 default) and fields
2. Parse args using loaded config for defaults
3. Print header
4. Parse CSV; print "Parsed CSV"
5. If --dry-run: print parsed values section (actual values from parsed data, not hardcoded) and exit 0 -- this is a stub; full computation added in Plan 02
6. Else: print "Live mode not yet implemented" and exit 1

Dry-run parsed values output format -- use f-strings with actual parsed values:
  Ordinary dividends (1099-DIV Box 1a):  ${div_1a:.2f}
  Qualified dividends (1099-DIV Box 1b): ${div_1b:.2f}
  Interest income (1099-INT Box 1):       ${int_box1:.2f}
  1099-B transactions: {count}
  Then one line per transaction showing term, date_acquired, date_sold, proceeds, cost, wash_sale, gain, form8949_code

Include if __name__ == '__main__': main() at the bottom.
